# 용어사전

본 문서는 현재 ABMS 코드베이스에서 실제로 사용 중인 유비쿼터스 언어를 정리합니다.
우선순위는 `도메인 코드 > 애플리케이션/어댑터 코드 > 화면 문구 > 기존 문서`입니다.

## 1. 핵심 기준 용어

| 기준 용어 | 코드/영문 | 의미 | 비권장/혼용 표현 |
|---|---|---|---|
| 직원 | `Employee` | 회사 소속 인력 | 인원 |
| 부서 | `Department` | 조직 단위 전반 | 팀(부서 전체를 뜻할 때) |
| 주관 부서 | `leadDepartmentId` | 프로젝트 손익이 귀속되는 부서 | 귀속팀 |
| 협력사 | `Party` | 프로젝트를 발주하거나 함께 일하는 외부 회사 | 거래처, 고객사 |
| 계정 | `Account` | 로그인과 권한의 주체 | 사용자 |
| 사용자명 | `username` | 계정 로그인 식별자. 현재는 이메일 형식 | 아이디 |
| 권한 | `Permission` | 특정 기능을 수행할 수 있는 권리 | 메뉴 권한 |
| 권한 그룹 | `PermissionGroup` | 여러 권한을 묶은 운영 단위 | 역할, Role |
| 권한 부여 | `GroupPermissionGrant` | 권한 그룹에 권한과 범위를 연결한 관계 | 권한 매핑 |
| 계정 그룹 할당 | `AccountGroupAssignment` | 계정에 권한 그룹을 연결한 관계 | 계정 권한 매핑 |
| 프로젝트 | `Project` | 계약, 투입, 손익의 기준 단위 | 사업 |
| 매출 계획 | `ProjectRevenuePlan` | 차수별 청구/매출 인식 계획 | 청구 계획, 수금 계획 |
| 프로젝트 투입 | `ProjectAssignment` | 직원이 프로젝트에 참여한 기간과 역할 | 프로젝트 할당, 배정 |
| 직원 원가 정책 | `EmployeeCostPolicy` | 연도와 고용형태별 원가율 규칙 | 비용 정책 |
| 월별 직원 원가 | `EmployeeMonthlyCost` | 특정 직원의 특정 월 원가 | 월 인건비 |
| 급여 이력 | `Payroll` | 직원의 연봉과 적용 기간 이력 | 급여 |
| 월별 손익 집계 | `MonthlyRevenueSummary` | 프로젝트별 월 매출/비용/이익 집계 | 월 마감 |
| 공통 코드 그룹 | `CommonCodeGroup` | 공통 코드의 분류 | 코드 마스터 |
| 공통 코드 상세 | `CommonCodeDetail` | 그룹 하위 개별 코드 | 상세 코드 |
| 알림 | `Notification` | 계정 대상 시스템 알림 | 메시지 |
| 채팅 세션 | `ChatSession` | 계정별 대화 묶음 | 대화방 |
| 채팅 메시지 | `ChatMessage` | 세션 안의 사용자/어시스턴트 발화 | 채팅 로그 |

## 2. 도메인별 용어

### 2.1 직원/조직

| 용어 | 코드/필드 | 설명 |
|---|---|---|
| 직원 ID | `employee.id` | 직원 식별자. 현재 대부분 `Long` 사용 |
| 소속 부서 | `employee.departmentId` | 직원이 속한 부서 |
| 직급 | `employee.position` | 조직 내 역할 수준 |
| 고용형태 | `employee.type` | 정직원/프리랜서/외주/반프리 |
| 재직상태 | `employee.status` | 재직, 휴직, 퇴사 |
| 등급 | `employee.grade` | 전문성 수준 |
| 아바타 | `employee.avatar` | UI 프로필 아바타 프리셋 |
| 부서장 | `department.leaderEmployeeId` | 해당 부서 리더 직원 |
| 상위 부서 | `department.parent` | 계층 구조 상 부모 부서 |
| 하위 부서 | `department.children` | 계층 구조 상 자식 부서 |

### 2.2 프로젝트/손익

| 용어 | 코드/필드 | 설명 |
|---|---|---|
| 프로젝트 코드 | `project.code` | 프로젝트 식별용 업무 코드 |
| 프로젝트 상태 | `project.status` | 예약, 진행 중, 완료, 보류, 취소 |
| 계약금액 | `project.contractAmount` | 프로젝트 총 계약 금액 |
| 프로젝트 기간 | `project.period` | 프로젝트 수행 기간 |
| 협력사 ID | `project.partyId` | 프로젝트 상대 협력사 |
| 주관 부서 ID | `project.leadDepartmentId` | 손익 귀속 부서 |
| 매출 계획 순번 | `projectRevenuePlan.sequence` | 프로젝트 내 차수 |
| 매출 예정일 | `projectRevenuePlan.revenueDate` | 매출 인식 기준 날짜 |
| 매출 유형 | `projectRevenuePlan.type` | 착수금/중도금/잔금/유지보수/기타 |
| 발행 여부 | `projectRevenuePlan.isIssued` | 세금계산서 발행 여부 |
| 투입 역할 | `projectAssignment.role` | DEV, PM, PL, ETC |
| 투입 기간 | `projectAssignment.period` | 직원의 프로젝트 참여 기간 |
| MM | `calculateManMonth()` | 월별 Man-Month 계산 결과 |
| 집계 기준일 | `summary.summaryDate` | 월별 집계 대상 일자 |
| 매출 금액 | `summary.revenueAmount` | 해당 월 확정 매출 |
| 비용 금액 | `summary.costAmount` | 해당 월 확정 비용 |
| 이익 금액 | `summary.profitAmount` | 매출 - 비용 |

### 2.3 권한/인증

| 용어 | 코드/필드 | 설명 |
|---|---|---|
| 로그인 계정 | `Account` | 직원과 1:1로 연결되는 인증 주체 |
| 계정 유효 여부 | `account.isValid` | 사용 가능 상태 |
| 로그인 실패 횟수 | `account.loginFailCount` | 인증 실패 누적 횟수 |
| 권한 코드 | `permission.code` | 예: `employee.read` |
| 권한 이름 | `permission.name` | 운영자가 보는 권한 명칭 |
| 권한 그룹 유형 | `groupType` | 시스템 기본 그룹 / 사용자 정의 그룹 |
| 권한 범위 | `PermissionScope` | 권한 적용 범위 |
| 등록 토큰 | `RegistrationToken` | 계정 등록 확인용 토큰 |

### 2.4 운영 지원

| 용어 | 코드/필드 | 설명 |
|---|---|---|
| 공통 코드 그룹 코드 | `groupCode` | 공통 코드 분류 키 |
| 공통 코드 | `CommonCodeDetailId.code` | 그룹 하위 코드 |
| 알림 유형 | `NotificationType` | 정보/성공/주의/오류 |
| 채팅 세션 ID | `sessionId` | 외부 AI 대화 세션 식별자 |
| 채팅 메시지 역할 | `ChatMessage.Role` | `USER`, `ASSISTANT` |
| 채팅 메모리 메시지 | `ChatMemoryMessage` | AI 메모리 저장용 메시지 |
