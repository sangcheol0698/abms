# ABMS 운영 관측성 플레이북

## API readiness 실패
- 먼저 `/actuator/health`와 `/actuator/health/readiness`를 확인한다.
- DB, 메일, OpenAI 중 어떤 컴포넌트가 down인지 본다.
- 최근 5분 `abms-api.log`의 `security-event`, `http-access`, `Exception occurred`를 확인한다.

## 5xx 비율 급증
- Grafana에서 `http_server_requests_seconds_count`의 5xx 비율을 본다.
- Loki에서 같은 시간대 `traceId`와 `exception` 필드로 검색한다.
- 최근 배포 여부와 외부 연동(OpenAI/메일) 실패율을 같이 본다.

## 인증 실패 급증
- `abms_security_authentication_failures_total` 추이를 확인한다.
- 특정 path 또는 source IP가 편중되는지 로그에서 확인한다.
- 로그인 실패와 세션 만료가 동시에 늘었는지 구분한다.

## 배치 실패
- `batch-event` 로그에서 `jobExecutionId`, `jobName`, `failureReason`을 확인한다.
- `abms.batch.job.executions.total`와 `abms.batch.job.last.failure.epoch`를 본다.
- 입력 파라미터(`targetDate`)와 실패 step을 먼저 확인한다.
